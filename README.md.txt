# Ego Taxi 🚕

![Ego Taxi Logo](https://via.placeholder.com/150x50.png?text=Ego+Taxi+Logo) <!-- החלף את הלינק עם הלוגו שלך -->

אפליקציית Ego Taxi היא פלטפורמה מתקדמת לחיבור בין נוסעים לנהגים בצורה חכמה, יעילה ובטוחה. האפליקציה כוללת ממשק משתמש אינטואיטיבי, מערכת ניהול נסיעות בזמן אמת, תשלומים מאובטחים ומערכת דירוגים ומשוב.

---

## תיאור הפרויקט 📝

### מטרה כללית
לפתח אפליקציית מוניות מתקדמת שתחבר בין נוסעים לנהגים בצורה חכמה, יעילה ובטוחה. האפליקציה תכלול:
- ממשק משתמש אינטואיטיבי.
- מערכת ניהול נסיעות בזמן אמת.
- תשלומים מאובטחים.
- מערכת דירוגים ומשוב.

---

## דרישות פונקציונליות 🛠️

### 1. משתמשים (Users)
#### נוסעים (Riders):
- יכולים להירשם ולהתחבר באמצעות אימייל/טלפון.
- יכולים להזמין נסיעה, לראות נהגים זמינים בסביבה, ולבחור נהג ספציפי.
- יכולים להגדיר שיטת תשלום (אשראי או מזומן).
- יכולים לראות את מסלול הנסיעה במפה ולעקוב אחר מיקום הנהג בזמן אמת.
- יכולים לדרג את הנהג לאחר הנסיעה ולהשאיר משוב.

#### נהגים (Drivers):
- יכולים להירשם ולהתחבר לאחר אישור מנהל המערכת.
- נדרשים להעלות 4 אישורים כתמונה (רישיון נהיגה, רישיון רכב, ביטוח, תעודת זהות) לאישור חד-פעמי.
- יכולים לראות הזמנות חדשות, לאשר או לדחות אותן.
- יכולים לראות את מסלול הנסיעה במפה ולקבל הוראות ניווט אוטומטיות (שילוב עם Waze/Google Maps).
- יכולים לראות פירוט נסיעות וסך ההכנסות שלהם.

### 2. ניהול נסיעות (Ride Management)
#### הזמנת נסיעה:
- הנוסע יכול להזמין נסיעה ולבחור נהג ספציפי.
- אם הנוסע לא בוחר נהג תוך 3 דקות, האפליקציה תשלח הזמנה אוטומטית לכל הנהגים הקרובים.
- הנהג הראשון שמאשר את ההזמנה מקבל אותה.

#### מעקב נסיעה:
- הנוסע והנהג יכולים לראות את מיקום הרכב בזמן אמת על המפה.
- הנהג מקבל הוראות ניווט אוטומטיות ליעד.

#### סיום נסיעה:
- לאחר סיום הנסיעה, האפליקציה מחשבת את העלות ומציגה אותה לנוסע.
- הנוסע יכול לשלם באמצעות אשראי או מזומן.
- לאחר התשלום, הנוסע יכול לדרג את הנהג ולהשאיר משוב.

### 3. ניהול תשלומים (Payments)
#### תשלום באשראי:
- פרטי האשראי של הנוסע נשמרים במערכת בצורה מאובטחת (הצפנה).
- התשלום מתבצע אוטומטית לאחר סיום הנסיעה.

#### תשלום במזומן:
- הנוסע משלם לנהג במזומן בסיום הנסיעה.

#### ניהול חשבוניות:
- בסוף החודש, הנוסע מקבל חשבונית עבור נסיעות עסקיות.
- הנהג מקבל פירוט הכנסות וחשבונית לצורך דיווחים.

### 4. מערכת דירוגים ומשוב (Rating & Feedback)
#### דירוג נהגים:
- הנוסע יכול לדרג את הנהג בסולם של 1-5 כוכבים.
- אם הדירוג נמוך מ-5 כוכבים, הנוסע יכול לרשום הערות לשיפור.

#### דירוג נוסעים:
- הנהג יכול לדרג את הנוסע ולהשאיר הערות.

#### ניהול תלונות:
- תלונות מועברות למרכז התמיכה של האפליקציה לטיפול.

### 5. ניהול פעילות (Activity Management)
#### רמת פעילות:
- רמת הפעילות של הנהג נקבעת לפי כמות הנסיעות שאושרו ובוטלו.
- נוסע שמבטל נסיעות ללא סיבה הוגנת נחסם לזמן מוגבל.

#### ביטול נסיעה:
- אם הנוסע מבטל נסיעה לאחר יותר מ-2 דקות, יחויב בתשלום של 12₪.
- אם הנוסע מבטל 3 נסיעות ברצף ללא סיבה הוגנת, חשבונו ייחסם למשך 10 דקות.

## דרישות טכניות 💻

### 1. טכנולוגיות
#### פיתוח Frontend:
- React Native או Flutter לפיתוח מובייל חוצה פלטפורמות (iOS ו-Android).

#### פיתוח Backend:
- Node.js עם Express.js או Python עם Django.
- מסד נתונים: Firebase Realtime Database או MongoDB.

#### שילוב API חיצוני:
- Google Maps API להצגת מפות וניווט.
- Waze API לניווט אוטומטי.
- Stripe או PayPal API לתשלומים מאובטחים.
- Twilio API לשליחת הודעות SMS.

### 2. אבטחה
- הצפנת נתונים רגישים (פרטי אשראי, מיקום).
- אימות דו-שלבי (2FA) עבור נהגים.
- הגנה מפני התקפות DDoS וזיוף נתונים.

### 3. תשתית
- שימוש ב-Cloud (AWS, Google Cloud, או Firebase) לאחסון וניהול נתונים.
- שילוב CDN לשיפור ביצועים.
